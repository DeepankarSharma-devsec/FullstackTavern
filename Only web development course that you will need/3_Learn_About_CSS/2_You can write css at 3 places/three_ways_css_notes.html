3 Ways to Write CSS

Understanding inline, internal, and external CSS

Introduction

Before jumping into styling, it's crucial to understand that visual appeal on a webpage isn't just about writing CSS. It begins with choosing the right fonts and images. Then comes the CSS.

  • Fonts shape the look and feel significantly.
  • Images guide the user's experience visually.
  • Components (buttons, cards, etc.) define user interactions.

The CSS Journey

CSS stands for Cascading Style Sheets, and its primary job is simple: Select elements and Style them. These styles cascade from parent to child elements.

Three Ways to Write CSS

1. Inline CSS

You can apply styles directly to an HTML element using the style attribute.

<h1 style="color: orange;">Welcome!</h1>

2. Internal CSS

CSS rules are placed within a <style> tag in the <head> section.

<style>
h2 {
    color: brown;
}
</style>

3. External CSS

CSS is stored in a separate file and linked using the <link> tag.

<link rel="stylesheet" href="styles.css">

This method enhances reusability and maintainability.

Using Classes and IDs

To target multiple elements with the same style, classes are used:

<p class="brown-text">Text here</p>

In CSS:

.brown-text {
    color: brown;
}

Specificity & Overrides

If multiple rules apply to the same element, CSS uses specificity rules to determine which one takes precedence.

  • Inline styles have the highest specificity.
  • IDs override classes.
  • Classes override element selectors.

Tip: Practice using all three methods and understand when to use each for best results!
