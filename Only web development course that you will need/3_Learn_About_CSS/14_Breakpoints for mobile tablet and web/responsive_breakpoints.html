📱 Responsive Design & CSS Breakpoints

A foundational guide to mastering media queries and building mobile-first designs.

🧠 Why Responsive Design?

  • Ensures your website works on mobile, tablet, and large screens.
  • Modern CSS & frameworks like Tailwind automate much of this.
  • Breakpoints control how your layout changes based on screen width.

🎯 Goals of This Lesson

  • Create a div that changes color based on screen size.
  • Use CSS media queries to define breakpoints.
  • Understand min-width vs max-width.

📐 Understanding Breakpoints

Min-Width: Minimum screen width required to apply the style.

Max-Width: Maximum screen width where style remains valid.

Think of them as job timings:

  • min-width: 601px = "I start work after 601px"
  • max-width: 600px = "I stop work after 600px"

💻 Sample Media Query Code

@media (max-width: 600px) {
  .responsive {
    background-color: #3498db;
  }
}

@media (min-width: 601px) and (max-width: 1024px) {
  .responsive {
    background-color: #e67e22;
  }
}

📊 Tailwind CSS Breakpoints (Reference)

  • sm: 640px
  • md: 768px
  • lg: 1024px
  • xl: 1280px
  • 2xl: 1536px

Tailwind simplifies breakpoints by offering pre-defined utility classes.
